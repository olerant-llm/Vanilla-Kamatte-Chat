🍦 バニラかまってチャット

構ってほしいので、AIがプッシュ型で話しかけてくるバニラチャット。

性格・口調・感情を設定したキャラクターが、ユーザーの入力がなくても数秒ごとに自分から話しかけてきます。放置するほど機嫌が悪くなる、ツンデレ仕様の単体HTMLチャットです。OpenAI互換APIに対応しています。


おてもとLLM の「バニラシリーズ」のひとつ。1ファイルで完結するフリーパーツ素材として、そのまま使ったり改造したりできます。




✨ 特徴


プッシュ型の自発発言 — ユーザーが黙っていると、キャラクターのほうから話しかけてきます。間隔は 6〜20 秒で自由に調整可能（初期値 9 秒）。
放置するほど不機嫌になる — 連続で無視されるほど「無関心に置かれている」感覚が強まり、怒りが指数関数的に蓄積します。構ってもらえると機嫌が直ります。
待ち時間を自覚する — 「最後の発言から約 N 秒待っている」という具体的な経過時間を踏まえて反応するので、「そんなに待ってないよ」に対して数字で言い返してきます。
感情パラメータ — 喜び・怒り・悲しみ・楽しさ・驚き・好意の6つを 0〜100 で管理。会話に応じて自動で増減し、その時の気分に合った態度・口調で返答します。
感情データがセリフに混ざらない設計 — AIには「まずセリフ、最後に [[EMO]] で感情の増減JSON」を出力させ、画面側でマーカー以降を切り離して処理します。数値が会話や履歴に残りません。
自由なキャラクター設定 — 名前・一人称・口調・性格を編集可能。初期値は素直になれないツンデレ「ミナ」。
長期記憶（メモリ） — 会話が一定量を超えると古いやり取りを自動で要約し、キャラクターが「覚えていること」として保持します。
OpenAI互換 — おてもとLLM / OpenAI / Gemini（OpenAI互換エンドポイント）/ 任意のカスタムエンドポイントに対応。SSEストリーミング、TTFT・トークン数などのメトリクス表示、生リクエストを確認できる開発者パネル付き。
単体HTML — 外部CSS・フォントに依存せず、1ファイルで画面が完結します。



🚀 使い方

1. サーバーで開く（推奨）

おてもとLLM以外のプロバイダや、署名付きリクエストを使う場合はローカルサーバー経由での起動を推奨します。同梱の start-local-server.bat（Windows / Python 3 が必要）を使うと簡単です。

textstart-local-server.bat をダブルクリック
 → 一覧から番号を選ぶ
 → ブラウザで http://localhost:8000/Vanilla_kamatte_chat.html が開きます

Python が入っていれば、手動でも起動できます。

bash# HTMLファイルがあるフォルダで実行
python -m http.server 8000
# → http://localhost:8000/Vanilla_kamatte_chat.html を開く

2. ファイルを直接開く

Vanilla_kamatte_chat.html をブラウザにドラッグ＆ドロップしても動きます（file://）。ただし、ブラウザによっては設定の保存やAPI接続（CORS）に制限が出ることがあります。


⚙️ 接続設定

「接続設定」タブから、利用するAPIを選びます。

プロバイダ必要な情報備考おてもとLLMAPI URL / Client Key / Server SecretHMAC-SHA256 署名付きリクエストOpenAIAPIキーブラウザから直接呼ぶとCORSで弾かれる場合ありGeminiAPIキーOpenAI互換エンドポイントを使用カスタムエンドポイントURL（＋任意のAPIキー）サーバー側でCORS許可が必要

設定や会話・感情はすべてブラウザの localStorage に保存されます（キーは otemoto_llm_chara_* で名前空間化）。サーバーには送信されません。


💬 自発発言モードの使い方

チャット画面右側の「💬 自発発言モード」をONにすると、プッシュ型の話しかけが始まります。


話しかける間隔 — 6〜20 秒で設定（初期値 9 秒）。範囲外の値は自動補正されます。
発火条件 — 送信中・タブが非表示・接続設定が未完了のときは、その回をスキップして次の間隔へ送ります（無駄打ち防止）。
カウントのリセット — ユーザーが返信したとき、モードをONにしたとき、会話をリセットしたときに、沈黙時間と連続無視カウントが0に戻ります。
停止 — 自発発言中も「停止」ボタンでその発言を中断できます。



⚠️ ONの間はアイドル状態でもAPIコールが発生します。トークン消費にご注意ください。




🎭 キャラクターと感情のしくみ

「キャラクター設定」タブで、名前・一人称・口調・性格・初期感情を編集できます。

AIへの出力ルールは次のとおりです。


まずキャラクターのセリフだけを書く（数値やト書きを混ぜない）
最後に必ず [[EMO]]{"joy":0,"anger":0,...} の形式で、今回の感情の増減量（各 -10〜+10）を書く


画面側は [[EMO]] 以降を吹き出しから取り除き、感情の更新だけに使います。表記が多少崩れても拾えるよう、寛容なパーサで処理しています。

放置時には、連続無視の回数に応じて怒りなどがコード側でも上乗せされます（小型モデルでも感情が確実に動くようにするため）。
